# 期貨商保證金一覽表

每天自動更新的「期貨商原始保證金一覽表」靜態網站。手機閱讀優先，最上方有字級調整與「可用保證金」即時換算可下單口數的功能。

資料來源：臺灣期貨交易所（TAIFEX）。第一階段僅含**國內**商品，國外交易所資料之後再加。

---

## 它怎麼運作（一分鐘理解）

1. `build.py`（純 Python 標準函式庫，**不需安裝任何套件**）會去 TAIFEX 抓四類保證金 CSV，以及每日收盤行情，產生一個 `index.html`。
2. GitHub Actions 依排程（台北 **08:00** 與 **20:00**）自動執行 `build.py`，把最新的 `index.html` 推回 repo。
3. GitHub Pages 把 `index.html` 當成網站對外服務。

整套機制和你現有的 `tw-market-dashboard` 一樣，沒有伺服器、不用維護，能放很久。

---

## 第一次設定（用 GitHub Desktop）

1. **建立 repo**：在 GitHub 網站新增一個 repo（例如 `tw-margin-table`）。
2. **加入檔案**：用 GitHub Desktop 把這個資料夾的內容（`build.py`、`README.md`、`.github/` 資料夾）放進你 repo 的本機資料夾，Commit 後按 **Push origin**。
   - `.github/workflows/update.yml` 一定要保持在這個路徑，Actions 才找得到。
3. **開啟 GitHub Pages**：到 repo 的 **Settings → Pages**，
   - Source 選 **Deploy from a branch**，
   - Branch 選 **main**、資料夾選 **/(root)**，存檔。
4. **先手動跑一次**：到 repo 的 **Actions** 頁籤 → 左側點 **更新保證金一覽表** → 右上 **Run workflow**。
   - 跑完後 repo 會多出 `index.html`（和除錯用的 `data.json`）。
   - 幾分鐘後你的網站就會在 `https://<你的帳號>.github.io/<repo 名稱>/` 上線。

之後就完全自動，不用再管。

---

## 排程說明

- GitHub 的 cron 使用 **UTC**：`0 0 * * *` = 台北 08:00、`0 12 * * *` = 台北 20:00。
- ⚠️ GitHub 免費排程在尖峰時段常有**數分鐘到數十分鐘的延遲**，這是 GitHub 的已知行為，不是程式出錯。若想立即更新，隨時可用 Actions 的 **Run workflow** 手動觸發。

## 保證金怎麼算的

- **股價指數類 / 商品類 / 匯率類**：直接取 TAIFEX 公告的「原始保證金」數字。
- **股票期貨**：保留「原始保證金適用比例」，用最新收盤價換算：

  | 類型 | 公式 |
  |---|---|
  | 一般股票期貨 | 收盤價 × **2,000** × 原始保證金適用比例 |
  | ETF 期貨 | 收盤價 × **10,000** × 原始保證金適用比例 |
  | 小型股票期貨（名稱含「小型」） | 收盤價 × **100** × 原始保證金適用比例 |

  > 小型契約那一條（×100）是我額外加的，因為小型股票期貨一口是 100 股，若沿用 ×2,000 會差 20 倍。請你確認這個處理是否符合你的需求；若不要，跟我說一聲即可移除。

收盤價只取**一般交易時段**、近月契約。

## 幣別

- TAIFEX 的保證金頁面沒有「幣別」欄位可程式化抓取，因此預設以 **新臺幣（TWD）** 計價，並針對已知的外幣商品另外標示：
  - 商品類：黃金期貨、黃金選擇權、美國原油期貨 → **USD**；臺幣黃金期貨 → TWD。
  - 匯率類：依報價幣別（兌美元→USD、兌人民幣→CNY、兌日圓→JPY、兌歐元→EUR）。
  - 指數類的國外指數期貨（道瓊、標普、那斯達克等）在期交所是以新臺幣計價，故標為 TWD。
- 「可下單口數」只針對 **TWD** 商品換算；外幣商品顯示「—」，避免未換匯的誤導數字。

## 檔案說明

| 檔案 | 用途 |
|---|---|
| `build.py` | 抓資料 + 產生 `index.html`。純標準函式庫。 |
| `.github/workflows/update.yml` | 自動排程設定。 |
| `index.html` | 產生的網站（執行後才會出現）。 |
